## Slide 5：COW 写时复制（通俗版）

**标题：** 创新点 1：COW 写时复制

**左边：笨办法**
```
你有一本书要给同事用
  ↓
复印一整本给他
  ↓
浪费纸张和时间
```

**右边：聪明办法（COW）**
```

```

**底部金句：**

## Slide 6：COW 写时复制（技术版）

**标题：** COW 实现原理

**流程图：**
```
fork 前：
  父进程页表 → [帧0, 帧1, 帧2]

fork 后：
  父进程页表 → [帧0, 帧1, 帧2]  ref_count = 2
  子进程页表 → [帧0, 帧1, 帧2]  ref_count = 2
  （共享同一份物理内存！）

子进程写入帧0 时：
  触发 COW → 复制帧0 → 帧3
  子进程页表 → [帧3, 帧1, 帧2]
  帧0 ref_count = 1
  帧3 ref_count = 1
  （只多了 4KB，而不是 12KB）
```

**设计建议：**
- 用箭头表示映射关系
- 用颜色区分共享帧和独占帧

## Slide 7：Stride 调度（通俗版）

**标题：** 创新点 2：Stride 调度

**比喻：操场跑步**

```
三个人在操场上跑步：
  - A 跑得快（优先级高）
  - B 跑得慢（优先级低）
  - C 跑得慢（优先级低）

不公平的调度：让 A 一直跑，B 和 C 饿着
我们的调度：A 跑 2 圈，B 和 C 各跑 1 圈

结果：A 跑了 50%，B 和 C 各跑了 25%
比例精确可控！
```

**底部金句：**
> "这就是 Linux CFS 调度器的数学基础。"

**设计建议：**
- 用跑道图表示
- A 用红色，B 和 C 用蓝色

## Slide 8：Stride 调度（技术版）

**标题：** Stride 算法原理

**公式：**
```
stride = BIG_STRIDE / (256 - priority)

进程 A: priority=10  → stride=4065（小步长，多 CPU）
进程 B: priority=133 → stride=8130（大步长，少 CPU）

调度规则：选 pass_value 最小的进程运行
运行后：pass_value += stride
```

**验证结果：**
```
200 个时间片后：
  A: cpu_time=99 (49.7%)  期望 50% ✓
  B: cpu_time=50 (25.1%)  期望 25% ✓
  C: cpu_time=50 (25.1%)  期望 25% ✓
```

**设计建议：**
- 左边公式，右边验证结果
- 用柱状图表示 CPU 时间比例

## Slide 10：管道 IPC（技术版）

**标题：** 管道实现原理

**流程：**
```
1. 创建管道：pipe() → read_fd=3, write_fd=4
2. fork 子进程：子进程继承文件描述符
3. 父进程写入：write(4, "Hello") → 数据进入管道
4. 子进程读取：read(3) → 从管道取出数据
```

**文件描述符表：**
```
父进程：fd=3(读端), fd=4(写端)
子进程：fd=3(读端), fd=4(写端)  ← 继承自父进程
```

**设计建议：**
- 用表格表示文件描述符
- 用箭头表示数据流向
