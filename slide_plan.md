## 第3段：COW 写时复制（3分钟）

### Slide 5：COW — 通俗解释

**左边：笨办法**
> 你有一本书要给同事用 → 复印一整本 → 浪费纸张

**右边：聪明办法（COW）**
> 把书给他看，告诉他"别写字"
> 如果他真的要写字，这时候再复印那一页
> 节省 90% 的纸张

> "这就是 Linux 内核 fork 高效的关键。"

### Slide 6：COW — 原理图

```
fork 前：  父进程 → [帧0, 帧1, 帧2]

fork 后：  父进程 → [帧0, 帧1, 帧2]  共享！
          子进程 → [帧0, 帧1, 帧2]  ref_count=2

写入后：  子进程 → [帧3, 帧1, 帧2]  只复制了帧0
```

### 🖥️ 现场演示 2：COW 验证

```bash
[PID:0] os> fork child
[PID:0] os> mem
  已使用: 4, 共享帧: 0

[PID:0] os> tick 5

[PID:0] os> mem
  已使用: 4, 共享帧: 4    ← 共享了！没有多用内存
```

> "fork 后共享帧从 0 变成 4，说明父子进程共享了同一份物理内存。
> 没有分配新帧，这就是零拷贝。"

### 🖥️ 现场演示 3：内存映射图

```bash
[PID:0] os> chart mapping
[图表] 内存映射图已保存: output/memory_mapping.png
```

**打开图片展示：**
> "这张图展示了虚拟页到物理帧的映射。
> 红色表示 COW 共享的帧，绿色表示独占的帧。"

## 第4段：Stride 调度（3分钟）

### Slide 7：Stride — 通俗解释

**操场跑步比喻：**
> 三个人在操场上跑步：
> - A 跑得快（优先级高）
> - B 和 C 跑得慢（优先级低）
>
> 不公平的调度：让 A 一直跑，B 和 C 饿着
> 我们的调度：A 跑 2 圈，B 和 C 各跑 1 圈
>
> 结果：A 跑了 50%，B 和 C 各跑了 25%

> "这就是 Linux CFS 调度器的数学基础。"

### Slide 8：Stride — 公式

```
stride = BIG_STRIDE / (256 - priority)

A: priority=10  → stride=4065（小步长，多 CPU）
B: priority=133 → stride=8130（大步长，少 CPU）

调度规则：选 pass_value 最小的进程运行
```

### 🖥️ 现场演示 4：Stride 公平性

```bash
[PID:0] os> fork A
[PID:0] os> nice 4 10        ← 高优先级

[PID:0] os> fork B
[PID:0] os> nice 5 133       ← 低优先级

[PID:0] os> fork C
[PID:0] os> nice 6 133       ← 低优先级

[PID:0] os> tick 100

[PID:0] os> ps
  A: cpu_time=50  (50%)
  B: cpu_time=25  (25%)
  C: cpu_time=25  (25%)
```

> "200 个时间片后，A 跑了 50%，B 和 C 各跑了 25%。
> 比例精确可控，和公式计算的结果一致。"

### 🖥️ 现场演示 5：公平性图表

```bash
[PID:0] os> chart stride
[图表] Stride 公平性图已保存: output/stride_fairness.png
```

**打开图片展示：**
> "这张柱状图直观地展示了 CPU 时间分配。
> A 的柱子是 B 和 C 的两倍，符合优先级比例。"

## 第5段：管道 IPC（2分钟）

### Slide 9：管道 — 通俗解释

**传纸条比喻：**
> 老板（父进程）要给员工（子进程）传文件
> 创建一个管道（就像两个人之间的通道）
> 老板把纸条放进去（write）
> 员工把纸条拿出来（read）

### Slide 10：管道 — 原理

```
1. 创建管道：pipe() → read_fd=3, write_fd=4
2. fork 子进程：子进程继承文件描述符
3. 父进程写入：write(4, "Hello")
4. 子进程读取：read(3) → "Hello"
```

### 🖥️ 现场演示 6：管道通信

```bash
[PID:0] os> pipe
管道已创建: read_fd=3, write_fd=4

[PID:0] os> fork child
[PID:0] os> write 4 "Hello from parent"
写入 18 字节: Hello from parent

[PID:0] os> read 3
读取 18 字节: Hello from parent
```

> "父进程写入的数据，通过管道传递给了子进程。
> 这就是操作系统最基础的进程间通信方式。"

## 演示节奏要点

1. **每一页 Slide 后紧跟实操** — 不要连续讲多页 Slide
2. **操作后停顿 2-3 秒** — 线上有延迟，让评委看清楚
3. **边操作边解释** — 不要沉默操作
4. **看摄像头** — 不是看屏幕，让评委感觉你在看他们
5. **遇到错误不慌** — "这个边界情况我们后续优化"
6. **语速放慢** — 线上答辩的关键
