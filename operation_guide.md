## ▌演示 1：Fork + ps + tree（~1.5min）

### 打開終端，啟動程序

```bash
cd process-management
python main.py
```

**等待啟動畫面出現，看到 `[PID:0] os>` 提示符。**

### 敲第一條命令：fork

```
[PID:0] os> fork shell
```

**預期輸出**：
```
子进程已创建: PID=1
```

**說**：
> fork 就像 Linux 的 fork 系統調用，創建一個子進程。
> 父進程拿到子進程的 PID=1。

### 敲第二條命令：再 fork 一個

```
[PID:0] os> fork editor
```

**預期輸出**：
```
子进程已创建: PID=2
```

### 敲第三條命令：ps 查看進程列表

```
[PID:0] os> ps
```

**預期輸出**：
```
PID    PPID   名称           状态         优先级    CPU时间
------------------------------------------------------------
0      -1     init           READY        255      0.0      *
1      0      shell          RUNNING      128      5.0
2      0      editor         READY        128      0.0

  总计: 3 个进程
```

**說**：
> ps 命令列出所有進程。可以看到 init 是根進程，shell 和 editor 都是它的子進程。
> 狀態、優先級、CPU 時間都和 Linux 的 ps 命令對應。

### 敲第四條命令：tree 看進程樹

```
[PID:0] os> tree
```

**預期輸出**：
```
进程树:
└── PID=0
    ├── PID=1
    └── PID=2
```

**說**：
> 進程樹清晰展示了父子關係。init 在根節點，兩個子進程掛在下面。
> 這和 Linux 的 pstree 命令一樣。

### 敲第五條命令：waitpid 回收

```
[PID:0] os> wait -1
```

**預期輸出**：
```
子进程退出，exit_code=0
```

**說**：
> wait -1 表示等待任意子進程退出。
> 父進程回收了子進程的退出碼，殭屍進程被清理。

### 再看一次進程列表

```
[PID:0] os> ps
```

**預期輸出**：
```
PID    PPID   名称           状态         优先级    CPU时间
------------------------------------------------------------
0      -1     init           READY        255      0.0      *
1      0      shell          RUNNING      128      5.0

  总计: 2 个进程
```

**說**：
> editor 已經被回收，進程列表少了一個。
> 這就是完整的 fork → exec → waitpid 生命週期。

**要點**：每敲一條命令，停 1 秒讓評委看到輸出。邊敲邊解釋，不要沉默操作。

## ▌演示 2：COW 驗證（~1.5min）

### 切回 VSCode

**操作**：按 Esc → overview → 點 VSCode。

### 創建子進程，查看內存

```
[PID:0] os> fork child
```

**預期輸出**：
```
子进程已创建: PID=1
```

```
[PID:0] os> mem
```

**預期輸出**：
```
内存状态:
  总帧数: 256
  已使用: 4
  空闲: 252
  共享帧: 0
```

**說**：
> fork 之前，已使用 4 幀，共享幀為 0。

### 推進時鐘，觸發 COW

```
[PID:0] os> tick 5
```

**預期輸出**：
```
已推进 5 个时钟周期
```

```
[PID:0] os> mem
```

**預期輸出**：
```
内存状态:
  总帧数: 256
  已使用: 4
  空闲: 252
  共享帧: 4
```

**說**：
> 注意看！已使用還是 4 幀，但共享幀從 0 變成了 4。
> 這說明父子進程共享了同一份物理內存，沒有額外分配。
> 零拷貝，零額外開銷。

**要點**：用手指著「共享幀: 4」這個數字，停 2 秒讓評委看清楚。

### 觸發寫入，觀察 COW 複製

```
[PID:0] os> write 4 "test cow"
```

**預期輸出**：
```
写入 8 字节: test cow
```

```
[PID:0] os> mem
```

**預期輸出**：
```
内存状态:
  总帧数: 256
  已使用: 5
  空闲: 251
  共享帧: 3
```

**說**：
> 寫入之後，已使用從 4 變成 5，共享幀從 4 變成 3。
> 只複製了被修改的那一幀，其餘三幀繼續共享。
> 這就是 COW 的精髓：按需複製，節省內存。

## ▌演示 4：管道 IPC（~1min）

### 繼續在 VSCode 中操作

```
[PID:0] os> pipe
```

**預期輸出**：
```
管道已创建: read_fd=3, write_fd=4
```

**說**：
> 創建一個管道，read 端是 fd=3，write 端是 fd=4。

```
[PID:0] os> fork child
```

**預期輸出**：
```
子进程已创建: PID=4
```

```
[PID:0] os> write 4 "Hello from parent"
```

**預期輸出**：
```
写入 18 字节: Hello from parent
```

**說**：
> 父進程往 fd=4 寫入數據。

```
[PID:0] os> read 3
```

**預期輸出**：
```
读取 18 字节: Hello from parent
```

**說**：
> 子進程從 fd=3 讀出了同樣的數據。
> 這就是 Linux 最基礎的進程間通信方式——管道。
> 數據在內核緩衝區裡傳遞，父子進程通過文件描述符讀寫。

**要點**：強調「父寫子讀」的方向性。

## ▌演示 6：級聯退出 + 孤兒回收（~1min）

### 創建多級進程樹

```
[PID:0] os> fork A
```

**預期輸出**：
```
子进程已创建: PID=5
```

```
[PID:0] os> tree
```

**預期輸出**：
```
进程树:
└── PID=0
    ├── PID=1
    ├── PID=2
    ├── PID=3
    ├── PID=4
    └── PID=5
```

**說**：
> 當前有 5 個子進程。

### 殺死一個進程，觀察級聯退出

```
[PID:0] os> kill 1
```

**預期輸出**：
```
进程 1 已被杀死
```

```
[PID:0] os> tree
```

**說**：
> 殺死進程 1 後，它的子進程（如果有的話）會成為孤兒。
> 系統自動把孤兒重新掛到 init 下，這就是孤兒回收機制。
> 和 Linux 的 reparent_orphans 原理一致。
